# Repository notes for Claude

## Running tests

**Always use `./test.sh`, not raw `pytest`.**

`./test.sh` runs `./lint.sh` first, then `pytest --cov=pirlygenes/ --cov-report=term-missing tests`.
`pyproject.toml` sets `addopts = "-n auto"`, so xdist spawns one worker per
logical CPU — fine here because the test suite is small (~68 tests) and
none of the curated CSVs are large. Memory pressure isn't an issue the way
it is in trufflepig.

Pass extra pytest args after the script name:

    ./test.sh -q
    ./test.sh tests/test_qc_feature_groups.py
    ./test.sh -x -v

## Package boundary (5.0+)

This package is **gene-knowledge data only**. Analysis, plotting, expression
matrices, and the `analyze` CLI moved to
[`trufflepig`](https://github.com/pirl-unc/trufflepig) in v5.0.

Things that belong here:
- Curated gene-set CSVs (therapy targets, CTAs, lineage panels, key-genes,
  cancer-type registry, narrative gene sets, rule tables, …)
- Symbol/ENSG resolvers (`gene_ids.py`, `gene_names.py`)
- Derived QC feature panels (`qc_feature_groups.py` + `data/qc-*.csv`)
- Cohort-baseline constants (`TCGA_MEDIAN_PURITY`)

Things that **don't** belong here (route to trufflepig):
- Expression matrices (pan-cancer, TCGA-deconv, subtype-deconv, HPA cell-type,
  matched-normal, ESTIMATE signatures)
- Normalization code (`normalize_expression`, `tpm_to_housekeeping_normalized`, …)
- Anything that runs an analysis on a sample

If you're tempted to add analysis logic here, push back — the migration was
deliberate.

## ENSG ID convention

ENSG IDs in `data/qc-*.csv` (and any other ENSG column you add) are stored
**unversioned** (`ENSG00000251562`, not `ENSG00000251562.5`). Strip the
`.<n>` suffix at the boundary. `pirlygenes.qc_feature_groups._strip_version`
is the canonical helper.

Symbols are uppercased on read.

## Regenerating derived data

The QC ENSG panels in `data/qc-*.csv` are **derived**, not curated — they're
generated by `scripts/generate_qc_gene_sets.py` walking every installed
GRCh38 Ensembl release and applying the regex from
`trufflepig.expression_qc.classify_gene_qc` as the seed. Re-run after the
regex panel changes:

    python scripts/generate_qc_gene_sets.py
