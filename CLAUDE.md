# Repository notes for Claude

## Running tests

**Always use `./test.sh`, not raw `pytest`.**

`./test.sh` runs `./lint.sh` first, then `pytest --cov=pirlygenes/ --cov-report=term-missing tests`.
`pyproject.toml` sets `addopts = "-n auto"`, so xdist spawns one worker per
logical CPU â€” fine here because the test suite is moderate (~130 tests
as of 5.1.0) and the only sizeable CSVs are the expression matrices
(~36 MB total). Memory pressure isn't an issue the way it is in
trufflepig.

Pass extra pytest args after the script name:

    ./test.sh -q
    ./test.sh tests/test_gene_families.py
    ./test.sh -x -v

## Package boundary (5.1+)

This package is **curated reference data + the mechanical operations on
it**. Analysis-layer code (CLI, plotting, sample-QC narration,
deconvolution, signature scoring) lives in
[`pirl-trufflepig`](https://github.com/pirl-unc/trufflepig).

Things that belong here:
- Curated gene-set CSVs (therapy targets, CTAs, lineage panels, key-genes,
  cancer-type registry, narrative gene sets, rule tables, â€¦)
- Symbol/ENSG resolvers (`gene_ids.py`, `gene_names.py`)
- Gene-family panels (`gene_families.py` + `data/{numt-pseudogenes,nuclear-retained-lncrnas,â€¦}.csv`)
- Reference expression matrices (`pirlygenes/expression/accessors.py` +
  `data/pan-cancer-expression.csv`, `tcga-deconvolved-expression.csv.gz`,
  `subtype-deconvolved-expression.csv.gz`, `hpa-cell-type-expression.csv`,
  `tumor-up-vs-matched-normal.csv`, `heme-tumor-up-vs-matched-normal.csv`,
  `estimate-signatures.csv`)
- Mechanical transforms on expression matrices
  (`pirlygenes/expression/normalize.py`: `normalize_expression`,
  `fpkm_to_tpm`, `renormalize_to_million`, `tpm_to_housekeeping_normalized`,
  `normalize_technical_rna_columns`, `normalize_technical_rna_long_table`)
- The QC classifier (`pirlygenes/expression/qc.py`:
  `classify_gene_qc`, `GeneQcClass`, `is_rescue_feature`)
- Transcriptâ†’gene rollup (`pirlygenes/expression/aggregate.py`)
- Cohort-baseline constants (`TCGA_MEDIAN_PURITY`)

Things that **don't** belong here (route to trufflepig):
- Per-sample QC narration (`summarize_qc_class_shares`,
  `top_gene_concentration`, `raw_qc_profile`,
  `technical_rna_component_phrase`, etc.)
- Library-prep / scale auto-detection (`apply_expression_qc_rescue`,
  `_expression_scale_qc`, `_detect_and_convert_to_tpm`)
- Anything that does hypothesis testing, statistical inference, or
  model fitting on a sample

The dividing line: "is this a transformation step on the data, or is
it a *judgment* about a sample?" Transformations belong here;
judgments belong in trufflepig.

5.0.x briefly drew the boundary tighter (no expression data, no
normalization) â€” that broke downstream consumers like topiary that
relied on the data + obvious-transforms API. 5.1.0 restored both;
this section reflects the post-correction layout.

## ENSG ID convention

ENSG IDs in the gene-family CSVs (`data/*-genes.csv`,
`data/*-pseudogenes.csv`, `data/nuclear-retained-lncrnas.csv`,
`data/immune-receptor-segments.csv`, `data/small-noncoding-rnas.csv`)
and any other ENSG column are stored **unversioned**
(`ENSG00000251562`, not `ENSG00000251562.5`). Strip the `.<n>` suffix
at the boundary. `pirlygenes.gene_families._strip_version` is the
canonical helper.

Symbols are uppercased on read.

## Regenerating derived data

The gene-family CSVs (`numt-pseudogenes.csv`, `nuclear-retained-lncrnas.csv`,
`rrna-and-pseudogenes.csv`, `ribosomal-protein-genes.csv`,
`ribosomal-protein-pseudogenes.csv`, `small-noncoding-rnas.csv`,
`histone-genes.csv`, `hemoglobin-genes.csv`, `immune-receptor-segments.csv`)
are **derived**, not curated â€” they're generated by
`scripts/generate_gene_family_sets.py` walking every installed GRCh38
Ensembl release and applying the regex from
`pirlygenes.expression.qc.classify_gene_qc` as the seed. Re-run after the
regex panel changes:

    python scripts/generate_gene_family_sets.py

`mitochondrial-genes.csv` is **curated** (with a semantic `Role` column
distinguishing protein-coding, rRNA, and tRNA entries) and not touched
by the script.

## Test fixtures for the cancer-type registry

`pirlygenes.gene_sets_cancer.CANCER_TYPE_NAMES` is a registry-backed
dict-like view that caches its forward map (`{code: name}`) and a
reverse map (`{display_name_lower: code}`) on first access. Tests
that monkey-patch `load_dataset.get_data` to swap in a fixture
registry must drop the caches before the next access, otherwise
they'll see the original CSV:

    from pirlygenes.gene_sets_cancer import _clear_caches

    def test_with_fixture_registry(monkeypatch):
        monkeypatch.setattr("pirlygenes.gene_sets_cancer.get_data", fake_get_data)
        _clear_caches()
        # ... now CANCER_TYPE_NAMES / resolve_cancer_type see the fixture
        _clear_caches()  # restore at teardown if other tests run after

`_clear_caches` resets every registry-backed cache in the module; the
underlying `_CancerTypeNamesView` also exposes `clear_cache()` if a
test wants finer control.
