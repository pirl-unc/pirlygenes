Identifier contract for collapsed expression tables (shipped this session, all green):

- **Unique gene→protein (not folded):** key = real **ENSG**, name = HGNC symbol. Untouched.
- **Folded proteoform (≥2 cDNA/protein-identical ENSGs summed):** member ENSGs LEAVE the key space; the merged row is keyed (both `Ensembl_Gene_ID` AND `Symbol`) by a **proteoform ID** = merged member symbols. Built by `pirlygenes.expression.protein_groups.proteoform_id`: shared prefix factored out + suffixes slashed (`XAGE1A`+`XAGE1B`→`XAGE1A/B`, `CT47A1..12`→`CT47A1/2/.../12`), full-slash if no shared prefix (`MAGEA9/MAGEA9B`), single name if identical. **Unique by construction** → split families like `CT45A` get distinct `CT45A2/8/9` vs `CT45A5/6/7` with no collision logic.

Key rules:
- Collapse only relabels a row when **≥2 loci were genuinely present and summed** in that context (`protein_groups.collapse_protein_identical_loci_long`, the matrix path); single-present-member rows keep their ENSG. That's the "summed across ENSGs" definition + the no-op test.
- `proteoform_id` replaced an earlier (rejected) "proteoform stem" approach (`XAGE1`) — stems collided across split families and needed a guard; the slash form is cleaner.
- **Display:** `gene_names.display_name` is the single id→name authority for symbols AND proteoform IDs. `CTAG1A/B`→NY-ESO-1, `CT47A1/.../12`→CT47A live in `gene_names.proteoform_display_aliases` — a SEPARATE map from `aliases`, because `aliases` feeds the bidirectional mapping synonym pool (`gene_mapping._curated_display_candidates`) and a synthetic ID must not compete with the real locus (CTAG1B) for reverse `NY-ESO-1→official` resolution. `XAGE1A/B` has no alias → displays as-is (fine).
- Both group CSVs (`data/{protein,cdna}-identical-gene-groups.csv`) regenerated from `scripts/generate_{protein,cdna}_identical_groups.py` (release 114); membership unchanged, only `group_canonical_symbol` (now the proteoform ID). The CT47A curated override (`proteoform-collapse-overrides.csv`) `group_symbol` also updated.
- Matrix now exposes ~86 proteoform-ID columns. `fold_to_cdna_canonical_symbol(['CTAG1B'])`→`['CTAG1A/B']`.

Dual gene/proteoform identifiers (PR #443, commit 89a0701): `cancer_reference_expression` long table always carries `Ensembl_Gene_ID` (row key), `Proteoform_ID` (stable proteoform key = bridge), `Member_Ensembl_Gene_IDs` (";"-joined constituent ENSGs = gene view), `Symbol`. Same schema for gene frame (`collapse_cdna_identical=False`) and proteoform frame (`=True`). Matched fold APIs: `fold_to_cdna_canonical_symbol` (→Symbol) + `fold_to_cdna_canonical_id` (→Ensembl_Gene_ID). CTA scripts use `_cta_ensgs()` (ENSG path, robust to symbol renames) over `_cta_symbols()`. Lesson: changing the key to a proteoform ID broke ENSG-keyed consumers (cta_covering_set/cta_expression_heatmaps selected by `CTA_gene_ids()`) — always provide both keys.

Consumer-facing folded panels (commit c27d650): `gene_sets_cancer.CTA_proteoform_symbols(kind)` / `CTA_proteoform_ids(kind)` expose the CTA panel pre-folded so consumers don't re-fold raw `CTA_gene_names()` against folded data (which silently drops clustered CTAs — NY-ESO `CTAG1A/B`, CT47A, …). **kind matters**: `"cdna"` (collapse_cdna_identical, matrix default) vs `"protein"` (collapse_protein_identical) DIFFER on protein-identical-but-cDNA-distinct loci (CGB3/5/8 vs CGB5/8). trufflepig folds by **protein** (`common.fold_panel_symbols` → protein_identical_groups); `CTA_proteoform_symbols(kind="protein")` is byte-equal to it. trufflepig is otherwise compatible (never requests pirlygenes collapse — does its own, keeping real ENSGs). `fold_to_protein_canonical_id` = protein analog of `fold_to_cdna_canonical_id`.

Display convention — structural in data, display name in labels (commit 601d386, v5.22.90): the library *declared* `display_name` (CTAG1B AND folded CTAG1A/B both → NY-ESO-1) but no `analyses/` script called it, so plots/CSVs leaked the raw proteoform ID and the alias was dead. Now ONE rule, applied uniformly: **data / matrix columns / CSV headers keep the STRUCTURAL id** (proteoform ID `CTAG1A/B` or raw symbol); **user-facing labels** (plot axes, annotations, markdown tables) route through `display_name`, re-exported as `analyses/_panels.py::display_label`, so the familiar antigen name shows (NY-ESO-1; CT47A). Un-aliased proteoform IDs (`XAGE1A/B`) pass through legibly. Applied at every antigen-label render site across the CTA plot scripts (cta_expression_heatmaps heatmap axes + summary.md, cta_patient_counts stacked bars/coverage curves/patient-count heatmaps/focus bar, cta_addressable_burden y-axes + console, cta_covering_set markdown). Verified: heatmap CSV header stays `CTAG1A/B` while summary.md / axes show `NY-ESO-1`. Rule of thumb: never CTAG1A/B and NY-ESO-1 interchangeably in the SAME layer — pick by layer (data=ID, display=name).

Centralized (commit 9ada23e): the cDNA/protein map+fold functions are now ONE `kind=`-parameterized core in protein_groups.py — `member_to_canonical(kind)`, `canonical_to_symbol(kind)`, `symbol_to_canonical(kind)`, `fold_symbols(panel, kind)`, `fold_ids(panel, kind)` (kind='cdna'|'protein'). All legacy per-space names (`fold_to_cdna_canonical_symbol`, `fold_symbols_to_canonical`, `cdna_member_to_canonical`, `protein_member_to_canonical`, …) are one-line aliases over it (trufflepig unbroken). Prefer the kind= API in new code. Known remaining asymmetry: collapse_cdna_identical/collapse_protein_identical (+ Proteoform_ID/Member_Ensembl_Gene_IDs columns) are ONLY on `cancer_reference_expression`, NOT on pan_cancer_expression/cancer_expression/hpa/estimate — downstream users of those fold manually.
